# Основные команды Git

Git — это система контроля версий, которая помогает нам отслеживать изменения в проекте и работать над кодом вместе с другими людьми. В этом уроке мы рассмотрим основные команды Git, которые вам понадобятся для работы с кодом.

---

1. Создание и инициализация репозитория
Теперь, когда Git настроен, давайте создадим репозиторий. Репозиторий — это место, где Git будет хранить все ваши изменения.

Инициализировать новый репозиторий:
git init
Эта команда создаёт новый репозиторий в текущей папке. Например, если вы работаете над проектом "Мой сайт", просто перейдите в эту папку и выполните git init. Теперь Git будет отслеживать все изменения.

Клонировать репозиторий:
Если вы хотите скачать репозиторий с GitHub или другого сервера, используйте команду:
git clone <repository_url>

Пример:
git clone https://github.com/username/repository.git
Эта команда создаст копию репозитория на вашем компьютере, чтобы вы могли начать работать с ним.

2. Работа с файлами
Git отслеживает все файлы в вашем проекте, и с помощью команд вы добавляете файлы для сохранения изменений.

Добавить файл в индекс:
git add <file>

Пример:
git add index.html
Эта команда добавляет файл index.html в "индекс" — подготовленный список файлов для следующего коммита.

Если вы хотите добавить все изменения, используйте команду:
git add .
Создать коммит:
Теперь, когда файл добавлен в индекс, можно зафиксировать изменения с помощью коммита. Коммит — это как сохранение игры, но в мире разработки.
git commit -m "Добавлен файл index.html"

Пример:
git commit -m "Добавлен начальный файл index.html"
Теперь изменения сохранены в репозитории, и все могут увидеть ваш прогресс.

Проверить статус:
git status
Эта команда покажет, какие файлы были изменены, добавлены или удалены, и готовы ли они к коммиту.

3. Работа с ветками
Ветки в Git позволяют работать над различными частями проекта одновременно. Например, можно работать над новой фичей, не мешая основной версии проекта.

Создать новую ветку:
git branch <branch_name>

Пример:
git branch feature
Эта команда создаёт новую ветку с именем feature. Теперь вы можете работать над новой фичей, не затрагивая основную ветку main.

Переключиться на ветку:
git checkout <branch_name>

Пример:
git checkout feature
Теперь вы переключились на ветку feature и можете вносить изменения в этой ветке.

Создать и переключиться на ветку одновременно:
git checkout -b <branch_name>

Пример:
git checkout -b new-feature
Эта команда создаёт новую ветку new-feature и сразу переключается на неё.

Слить ветки:
Когда работа над новой фичей завершена, нужно объединить её с основной веткой.
git merge <branch_name>

Пример:
git checkout main
git merge feature
Теперь изменения из ветки feature будут объединены с основной веткой main.

4. Синхронизация с GitHub
Теперь, когда вы вносите изменения в локальный репозиторий, нужно синхронизировать их с репозиторием на GitHub, чтобы другие могли увидеть ваши изменения.

Связать локальный репозиторий с удалённым:
git remote add origin <repository_url>

Пример:
git remote add origin https://github.com/username/repository.git
Теперь ваш локальный репозиторий связан с репозиторием на GitHub.

Отправить изменения на GitHub:
git push
Эта команда отправляет все изменения на GitHub.

Пример:
git push -u origin main
После первого пуша эта команда будет отправлять изменения в репозиторий на ветку main.

Получить изменения с GitHub:
Когда кто-то другой сделал изменения, нужно скачать их, чтобы синхронизировать ваш проект.
git pull

Пример:
git pull origin main
Эта команда загружает все изменения с GitHub и автоматически объединяет их с вашим локальным проектом.

5. Просмотр истории и изменений
Git позволяет легко отслеживать, что происходило в проекте. Например, можно посмотреть, кто и когда вносил изменения.

Просмотреть историю коммитов:
git log
Эта команда покажет список всех коммитов в проекте, с описаниями, авторами и датами.

Пример:
git log --oneline
Этот вариант покажет только краткую информацию о каждом коммите.

Просмотреть различия между версиями:
git diff
Команда покажет все изменения, которые ещё не были зафиксированы в коммите.

6. Полезные дополнительные команды
Эти команды полезны для исправления ошибок или управления репозиторием.

Отменить изменения в файле:
git restore <file>

Пример:
git restore index.html
Эта команда вернёт файл к состоянию, в котором он был на последнем коммите.

Удалить файл из индекса:
git reset <file>

Пример:
git reset index.html
Эта команда удалит файл из индекса, но оставит изменения на диске.

Пример рабочего процесса
Настройте Git:
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Создайте репозиторий:
git init
git remote add origin https://github.com/username/repository.git

Добавьте файл и сделайте коммит:
git add index.html
git commit -m "Добавлен начальный файл index.html"

Создайте ветку и отправьте изменения:
git checkout -b feature
git add .
git commit -m "Добавлена новая фича"
git push -u origin feature

Получите обновления с GitHub:
git checkout main
git pull origin main
